# AutoEdit - 验收评估标准

## 总则

每个 Phase 必须通过以下两类验收：

- **自动化验收**：通过测试脚本自动检测
- **人工验收**：由评估人员主观打分

所有 Phase 的验收必须 **同时通过两类** 才算完成。

## Phase 1：ASR + 文稿

### 自动化验收

| 编号 | 测试项 | 通过标准 | 测试方法 |
|------|--------|----------|----------|
| A1.1 | ASR 基本功能 | 输入音频，输出带时间戳的 JSON | `pytest tests/test_asr.py::test_basic_transcribe` |
| A1.2 | 词级时间戳 | 每个 segment 都有 words 字段 | `pytest tests/test_asr.py::test_word_timestamps` |
| A1.3 | 语气词检测 | 准确率 >= 85% | 100 个样本的 F1 score |
| A1.4 | 停顿检测 | 误报率 < 10% | 标注数据对比 |
| A1.5 | 违禁词检测 | 命中率 100%（词库内词全部识别） | 词库内词逐一测试 |
| A1.6 | 长音频处理 | 30+ 分钟音频分段处理成功 | `pytest tests/test_asr.py::test_long_audio` |
| A1.7 | 异常处理 | 无音频/损坏文件 -> 明确错误 | `pytest tests/test_asr.py::test_error_handling` |
| A1.8 | 耗时 | 30 分钟音频 ASR < 10 分钟 | 计时测试 |

### 人工验收

| 编号 | 检查项 | 通过标准 |
|------|--------|----------|
| M1.1 | ASR 准确率 | 抽查 5 分钟文稿，错字率 < 5% |
| M1.2 | 时间戳精度 | 抽查 10 个词，偏差 < 300ms |
| M1.3 | 语气词识别 | 抽查 20 个语气词，漏检 < 3 个 |
| M1.4 | 断句质量 | 句子切分合理，无断裂 |

### 评分方法

```
ASR 评分 = 准确率 * 0.4 + 时间戳精度 * 0.3 + 语气词检测 * 0.2 + 停顿检测 * 0.1
目标：>= 80 分
```

## 验收流程

```
每个 Phase 完成后：
  1. 开发者运行所有自动化测试 -> 全部通过
  2. 评估人员执行人工验收 -> 全部通过
  3. 两项都通过 -> Phase 完成
  4. 任一不通过 -> 修复后重新验收
```
